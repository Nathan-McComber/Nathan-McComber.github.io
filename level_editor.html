Redesigning a Puzzle Game Level Editor

As a freshman, I took a class about programming for game development. Me and 3 other students in the class were tasked with making a finished game in just one semester. We developed a puzzle game, and I created a level editor for our level designer using Windows Forms tools in Visual Studio. However, I pivoted from game development to UX design after freshman year and have learned a lot more about user interface design. As a result, I wanted to redesign the level editor as a personal project.

Primary Goal

Design a new level editor by applying what I learned about UI design to the original one.

Context for Game and Level Editor

The game we developed is titled Topping Tumble. It's a puzzle game where players place blocks, springs, and other items to guide pizza toppings from a fridge to an oven. Players have a limited amount of currency to spend on items for each level, and not every item can be used in every level.

The game can be downloaded here.

The editor I made aimed to make the level designer's job easier by providing a graphical representation of the level they're working on. It has the following features:

  • Adding/Removing Level Elements (walls, knives, buttons)
  • Modifying the amount of currency the player has
  • Modifying the number of toppings that come from the fridge
  • Specifying which items can be placed by the player
  • Saving a level to import into the game
  • Loading a level to modify it

Brainstorm/Sketch New Solutions

Before jumping straight into making a digital mockup for the level editor, I spent some time sketching out ideas on paper. I usually start with sketching because it allows me to try out multiple ideas without committing to one. The lack of commitment encourages me to explore more solutions and usually results in a better design overall.

Mockup the New Design

I took the sketches that I made and refined them into a “final” Level Editor by using the Windows Forms tools in Visual Studio. While I would normally use a tool like Figma to make a design mockup, I decided to use Visual Studio's Windows Forms tools to keep within the constraints of the original project.

Project Reflections

The new level editor fixes several issues that the original design had. It shows users all of the actions they can perform on one screen and makes the outcome of those actions obvious. It also provides hints with tooltips and prevents mistakes with warning dialogs.

Redesigning the old level editor helped me understand that:

Designs can always be improved.

When I initially designed the level editor, I thought that it was good enough. However, I found that many things needed to be changed after I assessed it for flaws. This highlights the importance of reviewing designs, since the first iteration will almost always have room for improvement.

Inputs should be obvious and have a clear purpose.

In my initial design for the level editor, users had to click on an icon of a tile to change its properties. It did not look like the icon was a button, and even if it was, the outcome of clicking on the button would be unclear. This was a major usability issue and was one of the main things I needed to change in my redesign.

Workflow steps should be kept in mind while designing.

I worked through the process of making a level to find problems in my initial editor design. If I had thought about the steps of making a level when I was originally designing the editor, I could have avoided some of the flaws I made in its design.
